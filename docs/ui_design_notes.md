# UI/UXデザイン詳細仕様

**最終更新**: 2025-12-24
**目的**: 要件定義書の補足資料（デザイン詳細）

## 全体デザイン原則

### 色使い（最大3色＋α）
```yaml
ベース色:
  - 白: #FFFFFF（背景）
  - ライトグレー: #F3F4F6（テーブル交互行、カード背景）
  - ミディアムグレー: #9CA3AF（テキスト、ボーダー）

アクセント色:
  - ブルー: #3B82F6（進行中、リンク、ボタン）
  - レッド: #EF4444（期限切れ、削除ボタン）
  - グリーン: #10B981（完了）※最小限使用

進捗ステータス色:
  - 未着手: グレー（#9CA3AF）
  - 進行中: ブルー（#3B82F6）
  - 完了: グリーン（#10B981）
  - 期限切れ: レッド（#EF4444）
```

### フォント
```yaml
和文: Noto Sans JP
  - 通常: 400
  - 太字: 700

英文: Inter
  - 通常: 400
  - 太字: 600

サイズ:
  - 見出し: 24px
  - サブ見出し: 18px
  - 本文: 14px
  - キャプション: 12px
```

### レスポンシブ対応
- MVP段階: PC優先（1280px以上）
- 最低限タブレット（iPad、768px以上）でも崩れないように
- モバイル対応は Phase 2 以降

## 共通コンポーネント

### ヘッダー（固定）
```yaml
位置: 上部固定
高さ: 64px
背景色: 白（#FFFFFF）
ボーダー: 下部に薄いグレー（#E5E7EB）

左側:
  - ロゴ/タイトル: 「ケアTaskFlow（仮）」
  - フォントサイズ: 20px、太字

右側:
  - ログアウトボタン
  - スタイル: テキストボタン（ブルー系）
```

### ナビゲーション
- サイドバーは不要（2ページしかないため）
- ヘッダーのみで十分

## P-001: 課題一覧ページ（Task List）

### レイアウト構成
```
┌─────────────────────────────────────────────────┐
│ ヘッダー（固定）                     [ログアウト] │
├─────────────────────────────────────────────────┤
│                                                 │
│  [カテゴリ▼] [進捗▼]              [+ 新規課題] │ ← フィルター＋新規ボタン
│                                                 │
│ ┌───────────────────────────────────────────┐   │
│ │ 項番 │ カテゴリ │ 問題点 │ 進捗 │ 対応者 │ 期限 │ ← テーブルヘッダー（固定）
│ ├───────────────────────────────────────────┤   │
│ │  1   │ レスパイト │ ○○○ │ 進行中 │ 田中 │ 2025-01-15 │ ← ホバーエフェクト
│ │  2   │ 学校     │ △△△ │ 未着手 │ 佐藤 │ 2025-02-01 │
│ │  3   │ 福祉サービス │ ××× │ 完了  │ 鈴木 │ 2024-12-20 │
│ └───────────────────────────────────────────┘   │
│                                                 │
└─────────────────────────────────────────────────┘
```

### フィルター部分
```yaml
配置: 上部（ヘッダー直下、テーブル上）
レイアウト: 横並び

カテゴリドロップダウン:
  - 幅: 200px
  - デフォルト: 「全てのカテゴリ」
  - 選択肢: 「３ 成人移行」「４ レスパイト」...等

進捗ドロップダウン:
  - 幅: 150px
  - デフォルト: 「全ての進捗」
  - 選択肢: 「未着手」「進行中」「完了」

動作:
  - リアルタイムフィルタリング（選択と同時に反映）
  - ボタン不要
```

### 新規登録ボタン
```yaml
配置: 右上（フィルター行の右端）
スタイル: プライマリボタン（ブルー系）
サイズ: 高さ40px、パディング左右24px
テキスト: 「+ 新規課題」
アイコン: プラスアイコン

動作:
  - クリックで課題詳細ページへ遷移（新規モード）
  - モーダル表示は不採用（詳細ページ遷移の方がシンプル）
```

### テーブル
```yaml
ヘッダー:
  - 固定: スクロールしても見える
  - 背景色: ライトグレー（#F3F4F6）
  - テキスト: 太字、ミディアムグレー（#6B7280）

列構成:
  - 項番: 60px（右寄せ）
  - カテゴリ: 150px
  - 問題点: 300px（可変、省略記号対応）
  - 進捗: 100px（色分けバッジ）
  - 対応者: 120px
  - 期限: 120px

行:
  - 高さ: 56px
  - 背景色: 交互（白 / ライトグレー）
  - ホバーエフェクト: 薄いブルー系背景（#EFF6FF）
  - クリック: 課題詳細ページへ遷移
  - カーソル: pointer

進捗ステータス（バッジ表示）:
  - 未着手: グレーバッジ
  - 進行中: ブルーバッジ
  - 完了: グリーンバッジ
  - 期限切れ: レッドバッジ（期限超過の場合）

ソート:
  - ヘッダークリックでソート
  - アイコン: ▲（昇順） / ▼（降順）
```

## P-002: 課題詳細ページ（Task Detail）

### レイアウト構成
```
┌─────────────────────────────────────────────────┐
│ ヘッダー（固定）                     [ログアウト] │
├─────────────────────────────────────────────────┤
│                                                 │
│  [← 戻る]  課題詳細               [編集] [削除] │ ← アクション
│                                                 │
│ ┌───────────────────┐ ┌───────────────────┐   │
│ │ 基本情報           │ │ 原因リスト         │   │
│ │                    │ │                    │   │
│ │ カテゴリ: [選択▼]  │ │ ┌───────────────┐ │   │
│ │ 問題点: [入力欄]   │ │ │ 原因1          │ │   │
│ │ 進捗: [選択▼]     │ │ └───────────────┘ │   │
│ │ 期限: [日付選択]   │ │ [+ 原因を追加]     │   │
│ │ 対応者: [選択▼]   │ │                    │   │
│ │                    │ │ 対応案リスト       │   │
│ │                    │ │ ┌───────────────┐ │   │
│ └───────────────────┘ │ │ 対応案1        │ │   │
│                        │ └───────────────┘ │   │
│                        │ [+ 対応案を追加]   │   │
│                        └───────────────────┘   │
│                                                 │
│                    [保存] [キャンセル]          │ ← 編集モード時のみ
│                                                 │
└─────────────────────────────────────────────────┘
```

### アクションボタン
```yaml
配置: 上部右側

戻るボタン:
  - 位置: 左上
  - スタイル: テキストボタン（← アイコン付き）
  - 動作: 課題一覧ページへ遷移

編集ボタン:
  - 位置: 右上
  - スタイル: セカンダリボタン（グレー系）
  - 動作: 編集モードに切り替え

削除ボタン:
  - 位置: 編集ボタンの右
  - スタイル: デンジャーボタン（レッド系）
  - 動作: 確認ダイアログ表示 → 削除実行
```

### 基本情報セクション（左側）
```yaml
カード形式:
  - 背景色: 白
  - ボーダー: 薄いグレー（#E5E7EB）
  - パディング: 24px
  - 幅: 40%

フィールド:
  - カテゴリ: ドロップダウン（必須）
  - 問題点: テキストエリア（必須、3行）
  - 進捗ステータス: ドロップダウン（必須）
  - 期限: 日付選択（任意）
  - 対応者: 複数選択ドロップダウン（任意）
  - 関連事業: テキストフィールド（任意）
  - 事業内容: テキストエリア（任意、2行）
  - 該当所属: テキストフィールド（任意）

編集モード:
  - 全フィールドが入力可能になる
  - 保存・キャンセルボタン表示
```

### 原因・対応案リスト（右側）
```yaml
カード形式:
  - 背景色: 白
  - ボーダー: 薄いグレー（#E5E7EB）
  - パディング: 24px
  - 幅: 55%

原因リスト:
  - 各原因をカード表示
  - カードスタイル: 薄いグレー背景、パディング12px
  - 削除アイコン: 右上（編集モード時のみ）
  - 追加ボタン: 「+ 原因を追加」（ブルー系テキストボタン）

対応案リスト:
  - 各対応案をカード表示
  - カードスタイル: 薄いグレー背景、パディング12px
  - 削除アイコン: 右上（編集モード時のみ）
  - 追加ボタン: 「+ 対応案を追加」（ブルー系テキストボタン）

編集モード:
  - 各カードがテキストエリアになる
  - 追加・削除が可能
```

### 保存・キャンセルボタン（編集モード時のみ）
```yaml
配置: 下部中央

保存ボタン:
  - スタイル: プライマリボタン（ブルー系）
  - サイズ: 高さ40px、幅120px
  - 動作: データベース保存 → 表示モードに戻る

キャンセルボタン:
  - スタイル: セカンダリボタン（グレー系）
  - サイズ: 高さ40px、幅120px
  - 動作: 変更を破棄して表示モードに戻る
```

### 削除確認ダイアログ
```yaml
タイトル: 「課題を削除しますか？」
メッセージ: 「この操作は取り消せません。本当に削除してよろしいですか？」
ボタン:
  - キャンセル: セカンダリボタン（グレー系）
  - 削除: デンジャーボタン（レッド系）
```
